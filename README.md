# ayuautobot

Бот для Telegram Business. Подключается к бизнес-аккаунту, ловит правки и
удаления сообщений собеседников, выполняет команды прямо из чата (сообщения
с командами удаляются автоматически) и работает сразу с несколькими
подключёнными аккаунтами — у каждого владельца свои независимые настройки.

Рассчитан на слабый сервер: Docker, 1 GB RAM, 1 ядро, 10 GB NVMe.

## Команды бота (личный чат с ботом)

Только четыре: **/start**, **/help**, **/settings**, **/admin**. Всё
управление — через inline-кнопки, без дополнительных команд.

- **/start** — приветствие и подключение бизнес-аккаунта.
- **/help** — подробное описание каждой функции с навигацией по темам.
- **/settings** — личные настройки (свои у каждого владельца):
  - 🔔 **Уведомления** — режим для удалений и правок отдельно: выключено /
    сразу / сразу без звука / копить и показывать по запросу (кнопка
    «📬 Показать уведомления» на главном экране), плюс переключатели
    «Абсолютно все удаленки» (показывать даже те удаления, что сделал сам
    бот — mute, `.view`) и «Сохранять одноразовые фото/гс/кружки».
  - 🧩 **Доп. функции** — Режим AFK (автоответ, текст настраивается отдельной
    кнопкой), Анонимные стикеры, Анти-спойлер, Антипоиск.
  - 🛠 **Команды** — включение/выключение каждой команды по отдельности.
  - 🗂 **Пресеты .say** — создание пресетов: имя, затем любое количество
    сообщений (текст/голосовые/кружки/фото/видео) одно за другим, кнопка
    «Готово» сохраняет пресет.
  - ⚙️ **Прочее** — TTL кэша, длительность mute по умолчанию.
- **/admin** (только для ID из `ADMIN_IDS`) — всё про сам бот:
  - 📦 Бэкапы — вкл/выкл, интервал, сколько хранить локально, сжатие.
  - 📥 Кэш и медиа — размер RAM-кэша, интервал очистки, квота на диск.
  - 💾 Данные — «сохранять вообще все сообщения в БД» (по умолчанию выключено:
    без этого тумблера в БД остаются только записи, нужные для детекции
    правок/удалений; полная история — по желанию).
  - 👥 Пользователи — список подключённых бизнес-аккаунтов.
  - 📤 Бэкап сейчас / 📢 Рассылка всем подключённым владельцам.

Все параметры в `/admin` редактируются **на лету**, без передеплоя —
хранятся в БД, `.env` используется только как стартовое значение при самом
первом запуске.

## Команды в чате с собеседником

| Команда | Что делает |
|---|---|
| `.spam N текст` | отправить текст N раз |
| `.spam N` (в ответ/reply на сообщение) | повторить N раз содержимое реплая — текст, **фото, видео, голосовое, кружок или стикер** |
| `.mute [сек]` / `.unmute` | удалять входящие сообщения собеседника (по умолчанию — значение из /settings) |
| `.typing N` | статус «печатает…» на N секунд |
| `.mock текст` | sPoNgEbOb-регистр |
| `.reverse текст` | текст задом наперёд |
| `.tr [язык] текст` | перевод (без языка — на русский) |
| `.qr текст` | QR-код |
| `.short ссылка` | короткая ссылка |
| `.id` / `.ping` | ID чата / проверка связи |
| `.say имя` | отправить пресет, настроенный в /settings |
| `.view N [текст]` | только на фото с такой подписью: через N секунд фото заменяется на чёрный квадрат, затем сообщение удаляется — обходит сохранение «последней версии» удалённых фото некоторыми клиентами |

Каждая команда включается/выключается индивидуально в /settings → 🛠 Команды.

## Архитектура (для слабого сервера)

```
Telegram Business API
        │
        ▼
   aiogram Dispatcher
        │
        ▼
     Storage ──────────► LRUCache (RAM, горячий слой, размер живой из /admin)
        │
        ▼
     SQLite (WAL) — owners / connections / messages / chat_stats /
                     say_presets / notifications_queue / global_settings
        │
        ▼
   BackupManager (по расписанию из /admin)
        ├─► консистентный snapshot через sqlite3 backup API
        ├─► gzip → документом в личку каждому админу
        └─► VACUUM + обрезка локальной истории до "горячего хвоста"
```

- **SQLite + WAL**, без Redis/Postgres — экономит память на 1 ГБ RAM.
- **RAM LRU-кэш** — большинство правок/удалений прилетают от Telegram в
  первые секунды, поэтому почти всегда не нужно даже читать с диска.
- **По умолчанию в БД не пишется вся история** — только то, что нужно для
  детекции правок/удалений в кэше. Полное сохранение всех сообщений —
  отдельный тумблер в /admin.
- **Бэкап-и-очистка** вместо бесконечного роста БД — снимок уезжает
  админу в ЛС, локально остаётся только «горячий хвост», диск не пухнет.
- **Без Pillow** — QR (`segno`) и чёрная заглушка для `.view` (собственный
  минимальный PNG-энкодер) работают без тяжёлых C-библиотек.

## Запуск

```bash
cp .env.example .env
# впишите BOT_TOKEN и свой ADMIN_IDS
docker compose up -d --build
docker logs -f ayuautobot-bot-1
```

Данные (БД, медиа) — в `./data` на хосте, volume монтируется в контейнер.
Entrypoint при старте автоматически чинит права на этот volume под
непривилегированного пользователя внутри контейнера.
