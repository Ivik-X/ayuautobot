# ayuautobot

Бот для Telegram Business. Подключается к бизнес-аккаунту, ловит правки и
удаления сообщений собеседников, выполняет команды прямо из чата (сообщения
с командами удаляются автоматически) и работает сразу с несколькими
подключёнными аккаунтами — у каждого владельца свои независимые настройки.

Рассчитан на слабый сервер: Docker, 1 GB RAM, 1 ядро, 10 GB NVMe.

## Команды бота (личный чат с ботом)

**/start**, **/help**, **/settings**, **/admin** — основное управление через
inline-кнопки. Плюс **/ghost** и **/link** — отдельные команды для режима
призрака (см. ниже), поскольку они могут вызываться со **второго**,
привязанного аккаунта, у которого нет доступа к /settings первого.

- **/start** — приветствие и подключение бизнес-аккаунта.
- **/help** — подробное описание каждой функции с навигацией по темам.
- **/ghost** — открыть список чатов в режиме призрака (см. раздел ниже).
- **/link КОД** — привязать текущий (второй) аккаунт как ghost-оператора по
  коду, выданному владельцем в /settings → 👻 Режим призрака.
- **/settings** — личные настройки (свои у каждого владельца):
  - 🔔 **Уведомления** — режим для удалений и правок отдельно: выключено /
    сразу / сразу без звука / копить и показывать по запросу (кнопка
    «📬 Показать уведомления» на главном экране), плюс переключатели
    «Абсолютно все удаленки» (показывать даже те удаления, что сделал сам
    бот — mute, `.view`) и «Сохранять одноразовые фото/гс/кружки».
  - 🧩 **Доп. функции** — Режим AFK (автоответ, текст настраивается отдельной
    кнопкой), Анонимные стикеры, Анти-спойлер, Антипоиск.
  - 🛠 **Команды** — включение/выключение каждой команды по отдельности.
  - 🗂 **Пресеты .say** — создание пресетов: имя, затем любое количество
    сообщений (текст/голосовые/кружки/фото/видео) одно за другим, кнопка
    «Готово» сохраняет пресет.
  - ⚙️ **Прочее** — TTL кэша, длительность mute по умолчанию.
- **/admin** (только для ID из `ADMIN_IDS`) — всё про сам бот:
  - 📦 Бэкапы — вкл/выкл, интервал, сколько хранить локально, сжатие.
  - 📥 Кэш и медиа — размер RAM-кэша, интервал очистки, квота на диск.
  - 💾 Данные — «сохранять вообще все сообщения в БД» (по умолчанию выключено:
    без этого тумблера в БД остаются только записи, нужные для детекции
    правок/удалений; полная история — по желанию).
  - 👥 Пользователи — список подключённых бизнес-аккаунтов.
  - 📤 Бэкап сейчас / 📢 Рассылка всем подключённым владельцам.

Все параметры в `/admin` редактируются **на лету**, без передеплоя —
хранятся в БД, `.env` используется только как стартовое значение при самом
первом запуске.

## Команды в чате с собеседником

| Команда | Что делает |
|---|---|
| `.spam N текст` | отправить текст N раз. Если N > 150 — команда всё равно мгновенно удаляется из чата, но отправка ждёт вашего подтверждения кнопкой в личном чате с ботом (10 минут на решение) |
| `.spam N` (в ответ/reply на сообщение) | повторить N раз содержимое реплая — текст, **фото, видео, голосовое, кружок или стикер** |
| `.mute [сек]` / `.unmute` | удалять входящие сообщения собеседника (по умолчанию — значение из /settings) |
| `.typing N` | статус «печатает…» на N секунд |
| `.mock текст` | sPoNgEbOb-регистр |
| `.reverse текст` | текст задом наперёд |
| `.tr [язык] текст` | перевод (без языка — на русский) |
| `.qr текст` | QR-код |
| `.short ссылка` | короткая ссылка |
| `.id` / `.ping` | ID чата / проверка связи |
| `.say имя` | отправить пресет, настроенный в /settings |
| `.view N [текст]` | только на фото с такой подписью: через N секунд фото заменяется на чёрный квадрат, затем сообщение удаляется — обходит сохранение «последней версии» удалённых фото некоторыми клиентами |
| `.watch` / `.unwatch` | включить/выключить слежение за именем, username и фото профиля в этом чате — уведомление при изменении |

Каждая команда включается/выключается индивидуально в /settings → 🛠 Команды.

## Дополнительно

- **📤 Экспорт переписки** (/settings → кнопка на главном экране) — JSON
  (структура в духе официального экспорта Telegram Desktop) и HTML в стиле
  мессенджера. Собирается только из того, что бот сам успел сохранить, пока
  был подключён — Bot API не даёт ретроактивного доступа к полной истории
  чата, поэтому это не замена официальному экспорту, а журнал с момента
  подключения бота. Для PDF — откройте HTML в браузере и «Печать → Сохранить
  как PDF». Для содержательного экспорта нужно, чтобы админ включил
  «сохранять все сообщения» в /admin → 💾 Данные — иначе в БД остаётся только
  то, что нужно для детекции правок/удалений.
- **🎙 Распознавание голосовых — локально** — расшифровывает гс/кружки/аудио,
  которые вы сами отправили боту **напрямую в личку** (не всё подряд в
  бизнес-чате). Работает через `faster-whisper` прямо на сервере, без внешних
  API и без затрат (`STT_ENABLED=true` в `.env`). Модель качается при первом
  использовании (`STT_MODEL_SIZE`, по умолчанию `base`, ~150 МБ) и держит
  часть RAM, пока бот запущен — для слабого сервера используйте `tiny`.
  Выключено по умолчанию, ничего не ломает.
- **👁‍🗨 Последние сообщения** (/settings → кнопка на главном экране) —
  быстрый просмотр последних N сообщений (10/20/50/100) из выбранного чата
  прямо текстом в личке с ботом, без скачивания файлов.
- **👁 Слежение за профилем** (`.watch`/`.unwatch`) — периодический опрос
  профиля собеседника (имя/username/фото), интервал задаётся в /admin.
  Telegram не шлёт push-события о смене чужого профиля, поэтому это именно
  опрос, а не мгновенное уведомление.

## Режим призрака (👻)

Позволяет читать и писать в чаты собеседников через бота — с основного
аккаунта или с привязанного второго — не открывая сам Telegram. Это **не**
второй логин/сессия аккаунта: бот как был Business Bot'ом на одном
аккаунте, так им и остаётся. «Второй аккаунт» просто получает право
командовать этим же ботом через свой собственный приватный чат с ним.

**Включение и привязка** (владелец, в /settings → 👻 Режим призрака):
1. Включить тумблер.
2. «Привязать второй аккаунт» — бот выдаёт одноразовый код (действует 5 минут).
3. Со второго аккаунта: открыть чат с этим же ботом, нажать /start, затем
   отправить `/link КОД`.
4. После 5 неверных попыток `/link` с одного аккаунта — блокировка на 10 минут
   (защита от подбора кода). Отвязать аккаунт можно в любой момент из /settings.

**Использование** — `/ghost` (у владельца или у привязанного оператора):
- Список чатов: закреплённые (закреплено в самом боте, это не то же самое,
  что закреп в Telegram — Bot API не даёт доступа к реальному списку
  закреплённых чатов аккаунта) — сверху, дальше по активности, у каждого в
  скобках число непрочитанных (тоже в терминах бота, см. ниже). Отдельная
  кнопка поиска по названию чата.
- При открытии чата — все непрочитанные сообщения + последние 10 (без
  дублей), с явной пометкой **🟢 Вы** / **🔵 Собеседник**. Под историей —
  кнопки «✅ Прочитать» (если есть непрочитанные), «📌 Закрепить/Открепить»,
  «⬅️ Список чатов».
- Пока чат открыт — любое сообщение, отправленное боту (текст, фото, голосовое,
  стикер), уходит прямо в этот чат от имени владельца. Новые сообщения
  собеседника, пока чат открыт, транслируются туда же в реальном времени.
- «✅ Прочитать» дополнительно вызывает `readBusinessMessage` — реальная
  метка «прочитано» проставляется и в самом Telegram, не только в боте.

**Честно про ограничения Bot API** (чтобы не было завышенных ожиданий):
- «Список чатов» — это чаты, которые бот уже видел (входящие/исходящие
  сообщения с момента подключения), а не полный список диалогов аккаунта —
  такого метода в Bot API просто нет.
- «Непрочитанные» и «закреплённые» — это собственная бухгалтерия бота, не
  синхронизированная с реальным состоянием клиента Telegram (кроме момента
  явного нажатия «Прочитать», которое дублируется в реальный аккаунт).
- Поиск ищет только среди уже известных боту чатов по названию — не по
  всей адресной книге Telegram.
- Работает достоверно только если у владельца одно бизнес-подключение;
  если их несколько, режим призрака берёт первое из них.
- Как и «Экспорт переписки», режим призрака нуждается в сохранённой
  истории — она начинает копиться автоматически, как только включён
  тумблер, независимо от общей настройки «сохранять все сообщения» в /admin.

## Про одноразовые (самоуничтожающиеся) медиа

Бот пытается скачать любое фото/гс/кружок сразу в момент получения апдейта,
с повторной попыткой. Если Telegram каждый раз отдаёт пустой `file_path` —
вероятная причина не в коде, а в гонке по времени: если получатель уже открыл
одноразовое медиа в своём клиенте до того, как апдейт дошёл до бота, файл
может быть уже уничтожен на серверах Telegram. Это не обязательно жёсткий
запрет Bot API, но и гарантии успеть тоже нет. Что можно сделать:
- включить **webhook-режим** (см. ниже) — так апдейты приходят от Telegram
  сразу, без задержки опроса, что поднимает шанс успеть скачать;
- если скачать всё же не удалось, бот больше не молчит — пришлёт в чат
  сообщение о неудачной попытке, вместо тишины.

## Webhook вместо polling (опционально)

По умолчанию бот получает апдейты через long polling — просто и не требует
домена. Если хочется меньшей задержки (актуально для одноразовых медиа),
можно включить webhook: нужен публичный HTTPS-домен, смотрящий на порт
`WEBAPP_PORT` (например, через nginx + Let's Encrypt). Задайте `WEBHOOK_URL`
в `.env` — при пустом значении бот работает как обычно, через polling.

## Архитектура (для слабого сервера)

```
Telegram Business API
        │
        ▼
   aiogram Dispatcher
        │
        ▼
     Storage ──────────► LRUCache (RAM, горячий слой, размер живой из /admin)
        │
        ▼
     SQLite (WAL) — owners / connections / messages / chat_stats /
                     say_presets / notifications_queue / global_settings
        │
        ▼
   BackupManager (по расписанию из /admin)
        ├─► консистентный snapshot через sqlite3 backup API
        ├─► gzip → документом в личку каждому админу
        └─► VACUUM + обрезка локальной истории до "горячего хвоста"
```

- **SQLite + WAL**, без Redis/Postgres — экономит память на 1 ГБ RAM.
- **RAM LRU-кэш** — большинство правок/удалений прилетают от Telegram в
  первые секунды, поэтому почти всегда не нужно даже читать с диска.
- **По умолчанию в БД не пишется вся история** — только то, что нужно для
  детекции правок/удалений в кэше. Полное сохранение всех сообщений —
  отдельный тумблер в /admin.
- **Бэкап-и-очистка** вместо бесконечного роста БД — снимок уезжает
  админу в ЛС, локально остаётся только «горячий хвост», диск не пухнет.
- **Без Pillow** — QR (`segno`) и чёрная заглушка для `.view` (собственный
  минимальный PNG-энкодер) работают без тяжёлых C-библиотек.

## Запуск

```bash
cp .env.example .env
# впишите BOT_TOKEN и свой ADMIN_IDS
docker compose up -d --build
docker logs -f ayuautobot-bot-1
```

Данные (БД, медиа) — в `./data` на хосте, volume монтируется в контейнер.
Entrypoint при старте автоматически чинит права на этот volume под
непривилегированного пользователя внутри контейнера.
