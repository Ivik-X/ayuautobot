# ayuautobot

Бот для Telegram Business («Автоматизация чатов от вашего имени»). Подключается
к бизнес-аккаунту через Telegram Business API, ловит правки/удаления
сообщений собеседников, пересылает их вам в личку и выполняет команды прямо
из чата с собеседником (сообщения-команды автоматически удаляются).

Эта версия переработана для:
- **мульти-пользовательской** работы — один бот-инстанс обслуживает
  сколько угодно бизнес-подключений, у каждого владельца свои настройки;
- **гибких настроек** через `/settings` (инлайн-меню) и `.set ключ значение`
  прямо в чате с собеседником;
- **новых полезных и не очень команд** (перевод, погода, курсы валют,
  QR-коды, сокращение ссылок, калькулятор, заметки, напоминания, AFK);
- работы на **слабом сервере** (Docker, 1 GB RAM, 1 ядро 3.3 GHz, 10 GB NVMe).

## Архитектура

```
Telegram Business API
        │
        ▼
   aiogram Dispatcher
        │
        ▼
     Storage  ──────────────►  LRUCache (RAM, горячий слой)
        │
        ▼
     SQLite (WAL) ── единая БД: owners / connections / messages /
                       chat_stats / notes / reminders
        │
        ▼
   BackupManager (раз в N часов)
        │
        ├─► консистентный snapshot через sqlite3 backup API
        ├─► gzip и отправка админу(ам) в личку документом
        └─► VACUUM + удаление локальной "холодной" истории
```

### Почему так

- **SQLite + WAL** вместо Redis/Postgres — на 1 ГБ RAM и 1 ядре отдельная
  СУБД съест память, которую лучше отдать самому боту. WAL даёт нормальную
  конкурентность чтения/записи без демона.
- **RAM LRU-кэш** — большинство правок/удалений прилетают от Telegram в
  первые секунды-минуты после отправки, поэтому попадание в кэш почти
  всегда экономит чтение с диска. Размер ограничен (`CACHE_MAX_ENTRIES`),
  вытеснение — по давности обращения.
- **Бэкап-и-очистка вместо бесконечного роста БД** — раз в
  `BACKUP_INTERVAL_HOURS` часов бот снимает консистентный снапшот БД (не
  останавливая приём сообщений), гзипует и отправляет админу в личные
  сообщения, после чего чистит локальную историю старше
  `BACKUP_KEEP_LOCAL_HOURS` и делает `VACUUM`, освобождая место на NVMe.
  Так 10 ГБ диска не переполнятся, а полная история всегда доступна вам
  в чате с ботом.
- **Медиа-квота** (`MEDIA_MAX_TOTAL_MB`) — при превышении лимита самые
  старые файлы удаляются автоматически.
- **Без Pillow** — QR-коды генерируются через чистый Python (`segno`),
  без тяжёлых C-библиотек обработки изображений.

## Мульти-пользовательность и гибкие настройки

Каждый владелец бизнес-подключения (определяется по Telegram user id) имеет
свой независимый профиль настроек, хранящийся в БД:

| Настройка | Что делает |
|---|---|
| `store_history` | писать ли историю переписки в БД (можно выключить ради приватности — тогда работает только горячий RAM-кэш) |
| `save_media` | скачивать ли медиафайлы локально |
| `notify_edit` / `notify_delete` | присылать ли уведомления о правках/удалениях |
| `feature_fun` | включает/выключает развлекательные команды |
| `feature_utility` | включает/выключает утилитарные команды (погода, перевод и т.д.) |
| `afk_enabled` / `afk_text` | автоответ собеседнику, пока вас нет |
| `cache_ttl_hours` | персональный TTL горячего кэша |
| `mute_default_seconds` | значение `.mute` без параметра |

Изменить их можно:
- в личном чате с ботом командой **`/settings`** (инлайн-кнопки);
- прямо в чате с собеседником: **`.set ключ значение`**, например
  `.set feature_fun off`.

## Админ-панель

Задайте свой Telegram user id в `ADMIN_IDS` (через запятую, если админов
несколько). В личном чате с ботом станут доступны:

- `/admin` — общая статистика (владельцы, подключения, размер БД/медиа,
  статус автобэкапа);
- `/users` — список подключённых бизнес-аккаунтов;
- `/backupnow` — сделать бэкап и схлопнуть историю немедленно, не дожидаясь
  расписания;
- `/broadcast текст` — разослать сообщение всем подключённым владельцам.

Именно админам бот отправляет файлы бэкапов БД (`.sqlite3.gz`) — как вы и
просили, чтобы не хранить всё на самом сервере.

## Команды в чате с собеседником

См. полный список через `.help` (или `/help` в личке с ботом) — он тоже
настраиваемый (`TEXT_HELP` / `TEXT_HELP_FILE`). Кратко:

**Модерация/автоматизация:** `.spam`, `.mute`, `.unmute`, `.typing`, `.purge`

**Развлечения:** `.dice`, `.flip`, `.8ball`, `.bomb`, `.love`, `.mock`,
`.clown`, `.owo`, `.reverse`, `.zalgo`, `.rand`

**Утилиты:** `.calc`, `.tr`, `.weather`, `.currency`, `.qr`, `.short`

**Заметки/напоминания/AFK:** `.note add|list|del`, `.say`, `.remind`, `.afk`

**Инфо:** `.stats`, `.chats`, `.info`, `.id`, `.ping`, `.settings`

## Запуск

```bash
cp .env.example .env
# впишите BOT_TOKEN и свой ADMIN_IDS
docker compose up -d --build
```

Данные (БД, медиа) лежат в `./data` на хосте — том смонтирован в контейнер.

## Переменные окружения

Смотрите подробные комментарии в `.env.example`. Всё, что не задано, берёт
разумные значения по умолчанию, рассчитанные на слабый сервер.
