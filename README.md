# AyuAuto

Бот для Telegram Business. Подключается к бизнес-аккаунту, ловит правки и
удаления сообщений собеседников, выполняет команды прямо из чата (сообщения
с командами удаляются автоматически) и работает сразу с несколькими
подключёнными аккаунтами — у каждого владельца свои независимые настройки.

Есть встроенная монетизация: бесплатный тариф с ограничениями, пробный
период и платная подписка через Telegram Stars — все лимиты и цена
настраиваются администратором на лету, без передеплоя.

## Как подключить бота к своему аккаунту

1. Откройте Telegram → **Настройки → Telegram для бизнеса → Чат-боты**
2. Нажмите **«Добавить бота»**
3. Введите username бота (бот сам покажет его в /start)
4. Нажмите **«Добавить»** и включите права: читать, отправлять и удалять сообщения

Отдельной deep-ссылки, которая сама открывает этот экран настроек с
предзаполненным именем бота, в Telegram не существует (это подтверждено
проверкой — ни один документированный `t.me`/`tg://` URL туда не ведёт),
поэтому шаги нужно проходить вручную один раз.

## Команды бота (личный чат с ботом)

- **/start** — приветствие и инструкция по подключению.
- **/help** — подробное описание каждой функции с навигацией по темам.
- **/settings** — **только уведомления**: режим для удалений и правок
  (выключено / сразу / сразу без звука / копить и показывать по запросу),
  переключатель «Абсолютно все удаленки» (показывать даже те удаления,
  что сделал сам бот — mute, `.view`).
- **/menu** — всё остальное:
  - 🛠 **Команды** — включение/выключение каждой команды по отдельности.
  - 🧩 **Доп. функции** — AFK, анонимные стикеры, антипоиск.
  - 🗂 **Пресеты .say** — создание пресетов (текст/голосовые/кружки/фото/видео,
    несколько сообщений подряд под одним именем).
  - 📤 **Экспорт последней истории** — JSON + HTML из того, что бот сам
    сохранил (не полный архив чата — см. пояснение ниже).
  - 👁‍🗨 **Последние сообщения** — быстрый просмотр последних N сообщений чата.
  - 👻 **Режим призрака** — включение, привязка второго аккаунта, и кнопка
    «Открыть чаты» (см. раздел ниже).
  - 💫 **Подписка** — статус, пробный период, оплата, промокод.
  - ⚙️ **Прочее** — TTL кэша, длительность mute по умолчанию.
- **/ghost** и **/link КОД** — для **второго**, привязанного аккаунта в
  режиме призрака: у него нет доступа к /menu первого аккаунта, поэтому это
  отдельные команды (подробнее ниже).
- **/admin** (только для ID из `ADMIN_IDS`) — всё про сам бот: бэкапы,
  кэш и медиа-квота, минимум свободного места на диске, цена и пробный
  период подписки, лимиты бесплатного тарифа, промокоды, список
  пользователей, бэкап по требованию, рассылка.

## Команды в чате с собеседником

| Команда | Что делает |
|---|---|
| `.spam N текст` | отправить текст N раз. Если N > 150 — команда всё равно мгновенно удаляется из чата, но отправка ждёт вашего подтверждения кнопкой в личном чате с ботом (10 минут на решение) |
| `.spam N` (в ответ/reply на сообщение) | повторить N раз содержимое реплая — текст, **фото, видео, голосовое, кружок или стикер** |
| `.mute [сек]` / `.unmute` | удалять входящие сообщения собеседника |
| `.typing N` | статус «печатает…» на N секунд |
| `.mock текст` | sPoNgEbOb-регистр |
| `.reverse текст` | текст задом наперёд |
| `.tr [язык] текст` | перевод (без языка — на русский) |
| `.qr текст` | QR-код |
| `.short ссылка` | короткая ссылка |
| `.id` / `.ping` | ID чата / проверка связи |
| `.say имя` | отправить пресет, настроенный в /menu |
| `.view N [текст]` | на фото с такой подписью: через N секунд фото заменяется на чёрный квадрат, затем сообщение удаляется |
| `.watch` / `.unwatch` | слежение за именем/username/фото профиля в этом чате |

Каждая команда включается/выключается индивидуально в /menu → 🛠 Команды.

## Подписка и монетизация

Три состояния: **бесплатный** тариф, **пробный период** (по умолчанию 3 дня,
активируется один раз, кнопка в /menu → 💫 Подписка) и **полный доступ**
(оплата звёздами или промокод). Всё, что ниже — стартовые значения по
умолчанию, админ меняет любое из них в /admin → 🚦 Лимиты бесплатного тарифа
и 💫 Подписка: цена и пробный период, без передеплоя.

**На бесплатном тарифе по умолчанию:**
- уведомления об удалении/правке приходят с **скрытым текстом** — открыть
  можно кнопкой, лимит открытий — 3 в месяц;
- пресетов `.say` — максимум 1;
- `.spam` — максимум 20 сообщений суммарно в месяц;
- `.mute` — максимум 180 секунд суммарно в месяц;
- `.view`, AFK, антипоиск, расшифровка голосовых, режим призрака —
  недоступны;
- анонимные стикеры, `.watch`, экспорт, просмотр последних сообщений —
  недоступны (один общий переключатель в /admin на всё это).

Все лимиты обнуляются 1-го числа каждого месяца автоматически.

**Промокоды** — админ создаёт в /admin → 🎟 Промокоды: тип `free_days`
(N дней полного доступа) или `discount` (% скидки на следующую оплату),
максимум использований и срок жизни кода. Пользователь вводит код в
/menu → 💫 Подписка → «Ввести промокод».

### Как подключить оплату звёздами (Telegram Stars)

Специальных внешних настроек не требуется — Stars обрабатываются самим
Telegram, без платёжного провайдера и без токена:

1. Никаких `PROVIDER_TOKEN` или регистраций не нужно — код уже использует
   валюту `XTR` (Telegram Stars) в `send_invoice`, для неё provider-токен
   не требуется в принципе.
2. Убедитесь, что у бота включён приём платежей: обычно это доступно
   сразу после создания бота через @BotFather; при проблемах проверьте
   там пункт **Payments** для своего бота.
3. Задайте цену в звёздах в /admin → 💫 Подписка → «Цена подписки» (по
   умолчанию 50 ⭐/месяц).
4. Готово — пользователь нажимает «⭐ Оплатить N звёзд» в /menu → 💫 Подписка,
   Telegram показывает системное окно оплаты звёздами, после успешной
   оплаты бот сам продлевает подписку на 30 дней и сбрасывает
   использованную скидку.

Комиссию и вывод звёзд Telegram обрабатывает на своей стороне (Fragment/
конвертация в TON) — это не часть этого бота.

## Режим призрака (👻)

Позволяет читать и писать в чаты собеседников через бота — с основного
аккаунта или с привязанного второго — не открывая сам Telegram. Это **не**
второй логин/сессия аккаунта: бот как был Business Bot'ом на одном
аккаунте, так им и остаётся. «Второй аккаунт» просто получает право
командовать этим же ботом через свой собственный приватный чат с ним.

**Включение и привязка** (владелец, в /menu → 👻 Режим призрака):
1. Включить тумблер (требует подписки — см. выше).
2. «Привязать второй аккаунт» — бот выдаёт одноразовый код (действует 5 минут).
3. Со второго аккаунта: открыть чат с этим же ботом, нажать /start, затем
   отправить `/link КОД`.
4. После 5 неверных попыток `/link` с одного аккаунта — блокировка на 10 минут.
   Отвязать аккаунт можно в любой момент из /menu.

**Открытие чатов** — кнопка «📂 Открыть чаты» в /menu → 👻 Режим призрака
(владельцу) или команда `/ghost` (второму, привязанному аккаунту — у него
нет доступа к /menu первого).
- Список чатов: закреплённые (закреплено в самом боте — это не то же самое,
  что закреп в Telegram, у Bot API нет доступа к реальному списку
  закреплённых чатов аккаунта) сверху, дальше по активности, у каждого в
  скобках число непрочитанных (тоже в терминах бота). Отдельная кнопка
  поиска по названию чата.
- При открытии чата — все непрочитанные сообщения + последние 10 (без
  дублей), с явной пометкой **🟢 Вы** / **🔵 Собеседник**. Под историей —
  кнопки «✅ Прочитать» (если есть непрочитанные), «📌 Закрепить/Открепить»,
  «⬅️ Список чатов».
- Пока чат открыт — любое сообщение, отправленное боту, уходит прямо в этот
  чат от имени владельца. Новые сообщения собеседника транслируются туда же
  в реальном времени.
- «✅ Прочитать» дополнительно вызывает `readBusinessMessage` — реальная
  метка «прочитано» проставляется и в самом Telegram.

**Честно про ограничения Bot API:** список чатов — это только то, что бот
уже видел, а не полный список диалогов аккаунта (такого метода в Bot API
нет); «непрочитанные» и «закреплённые» — собственная бухгалтерия бота, не
синхронизированная с реальным Telegram (кроме явного «Прочитать»); поиск
ищет только среди уже известных боту чатов; если у владельца несколько
бизнес-подключений, режим призрака работает с первым из них.

## Архитектура (для слабого сервера)

```
Telegram Business API
        │
        ▼
   aiogram Dispatcher
        │
        ▼
     Storage ──────────► LRUCache (RAM, горячий слой, размер живой из /admin)
        │
        ▼
     SQLite (WAL) — owners / connections / messages / subscriptions /
                     usage_counters / promo_codes / say_presets / ...
        │
        ▼
   BackupManager (по расписанию из /admin) + DiskGuard (аварийно, по месту на диске)
        ├─► консистентный snapshot через sqlite3 backup API
        ├─► gzip → документом в личку каждому админу
        └─► VACUUM + обрезка локальной истории
```

- Сообщения **всегда** пишутся в БД (без опционального тумблера) — иначе
  после рестарта контейнера теряется и RAM-кэш, и история одновременно.
  Диск от разрастания защищают два независимых механизма:
  - **Плановый бэкап** по расписанию (/admin → 📦 Бэкапы): снимок в личку
    админу, затем обрезка локальной истории до «горячего хвоста».
  - **Аварийный диск-guard**: если свободного места на диске меньше
    порога (по умолчанию 5 ГБ, /admin → 💾 Данные), бот сначала делает
    внеплановый бэкап (снимок — админу), затем удаляет самые старые записи
    пачками, пока место не освободится.
- **RAM LRU-кэш** — горячий слой перед SQLite, размер живой (/admin).
- **Без Pillow** — QR (`segno`) и чёрная заглушка для `.view` работают без
  тяжёлых C-библиотек. Локальное распознавание речи — `faster-whisper`
  (CPU, без внешних API и без затрат).

## Что было вырезано при переходе в продакшн

- **Сохранение одноразовых (самоуничтожающихся) медиа** — так и не удалось
  добиться стабильной работы через официальный Bot API (вероятная причина —
  гонка по времени с моментом уничтожения на серверах Telegram, а не баг в
  коде), решено не тащить ненадёжную функцию в продакшн.
- **Анти-спойлер** — не удалось гарантированно починить за несколько
  попыток без возможности живой отладки; убран, а не оставлен «работающим
  через раз».

## Запуск

```bash
cp .env.example .env
# впишите BOT_TOKEN и свой ADMIN_IDS
docker compose up -d --build
docker logs -f ayuauto-bot
```

Данные (БД, медиа, модели STT) — в `./data` на хосте, volume монтируется в
контейнер. Entrypoint при старте автоматически чинит права на этот volume
под непривилегированного пользователя внутри контейнера.

## Обновление на новом сервере

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo usermod -aG docker $USER && newgrp docker
```

Дальше — сам проект:

```bash
mkdir -p ~/ayuauto && cd ~/ayuauto
unzip ayuauto.zip -d .
cp .env.example .env
nano .env
docker compose up -d --build
```

Для последующих обновлений — простой скрипт `deploy.sh` в корне проекта:
останавливает контейнер, подтягивает актуальный код и поднимает заново.
